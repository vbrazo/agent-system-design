# Module 1: Agent Design Fundamentals

This module introduces the core concepts of agentic AI system design—from what makes a system an agent, to architecture, memory, orchestration, safety, and production readiness.

## Lessons

| # | Lesson | Topics |
|---|--------|--------|
| 1 | [Introduction to AI Agents](./01-introduction-to-ai-agents.md) | Perceive-reason-act, models vs. agents, agent types, evolution, when to build |
| 2 | [Core Agent Components](./02-core-agent-components.md) | Model, tools, instructions, modular design |
| 3 | [Component Interaction and Memory](./03-component-interaction-and-memory.md) | Architecture loop, sensing, short/long/external memory, RAG |
| 4 | [Agent Orchestration Patterns](./04-agent-orchestration-patterns.md) | Tool loops, ReAct, plan-and-execute, multi-agent patterns, frameworks |
| 5 | [Guardrails and Human Oversight](./05-guardrails-and-human-oversight.md) | Safety layers, HITL/HOTL/HATL, trustworthy design |
| 6 | [Challenges and Design Strategies](./06-challenges-and-design-strategies.md) | Latency, hallucination, scalability, security, production checklist |

## Learning outcomes

After completing this module, you will be able to:

- Distinguish AI agents from passive models and choose when agentic design is warranted
- Decompose an agent into model, tools, and instructions—and extend that with perception and memory
- Select orchestration patterns for single- and multi-agent systems
- Layer guardrails and human oversight for safe, trustworthy deployment
- Apply practical strategies for latency, reliability, scale, integration, and evaluation
